# A-10C

Full support: CDU display, LEDs, brightness, and a complete front panel feed (flight
data, gear, barometric pressure, and the AGP32 clock).

## Supported devices

| Device | What it shows |
|--------|---------------|
| CDU (MCDU / PFP3N / PFP7 / PFP4) | The A-10C CDU pages, optional CMSP overlay, status LEDs |
| FCU + EFIS | Heading, speed, altitude, vertical speed, barometric pressure (QNH on the EFIS) |
| PAP3 | Heading, speed, altitude, vertical speed |
| AGP32 | Landing-gear LEDs and the cockpit clock (chrono / UTC / ET) |

## CDU display

- All ten CDU lines (`CDU_LINE0`–`CDU_LINE9`) are mirrored to the device.
- **Bottom-aligned** option shifts the page down so it sits at the bottom of the screen.
- **CMS overlay** (optional): when enabled, the two CMSP lines are drawn above/below the
  page with an amber separator so you can watch the countermeasures program from the CDU.

### LED mapping

| CDU LED | DCS indicator |
|---------|---------------|
| FAIL | Master Caution |
| FM1 | Gun Ready |
| IND | Nosewheel Steering |
| FM2 | Canopy unlocked |

## Takeoff performance page

Calculations use the [A10C-PerfLib](https://github.com/landre-cerp/A10C-PerfLib)
(included as a submodule), assume **flaps 7**, and take QNH silently from the aircraft
altimeter (standard 29.92 inHg until it is read).

### Setup

1. **Enable "Takeoff Performance Page"** in the A-10C options.  
   Sets the key that opens the page — default is **FuelPred**; change it with the
   **Perf Page Key** combo box.

2. **Unbind the LSKs, digits, and slash in DCS.**  
   In the A-10C key bindings, remove the DCS assignments for the eight line-select keys
   (LSK 3L/5L/7L/9L and 3R/5R/7R/9R), the ten digit keys (0–9), and the slash.  
   Without this step those keys fire twice: once on the bridge page, once in the sim.

3. **Enable "Forward CDU keys to sim"** in the A-10C options.  
   After unbinding, the bridge takes over: while the **live CDU page** is shown it sends
   every LSK/digit/slash press to the real A-10C CDU over DCS-BIOS, so the in-game CDU
   keeps working normally. While the **takeoff page** is open the same keys feed the
   scratchpad and nothing is forwarded.

> **Independent toggles** — the two options are separate. You can run forwarding without
> the takeoff page (useful if you simply want the bridge to drive the CDU after unbinding),
> or keep the takeoff page without forwarding (if you leave DCS bindings in place and accept
> the double-fire on the live-CDU page). For full perf-calc use, enable both.

### Using the page

Press the **Perf Page Key** to open the page; press it again (or a page navigation key) to
return to the live CDU.

Enter values with the scratchpad and line-select keys; **RCR** and **taxi fuel** cycle on
each press (marked with `Δ`). Takeoff weight `TOW = GW − taxi fuel`.

| Key | Field | | Key | Field |
|-----|-------|-|-----|-------|
| L1 | Elevation (ft) | | R1 | Temperature (°C) |
| L2 | Runway length (ft) | | R3 | RCR (DRY/WET/ICY, cycle) |
| L3 | QFU (°) | | R5 | Taxi fuel (100/200/300, cycle) |
| L4 | Wind (`DIR/SPD`, e.g. `200/15`) | | | |
| L5 | Gross weight (lbs) | | | |

Read-outs (green): **PTFS**, **VR**/**TO** speeds, ground run (**GR**) and critical field
length (**CFL**). GR and CFL turn **red** when they exceed the entered runway length.

> The LSK mapping assumes the top four physical keys per side map to the A-10C's
> `3/5/7/9` line-select positions.

## Brightness / lighting

- **Display brightness** follows the CDU `DIM`/`BRT` keys (5% steps).
- **Backlight brightness** follows the left-console lighting rotary on the right pedestal.
- Disable everything with the global **"Disable lighting management"** option if you want
  SimAppPro (or the aircraft alone) to drive brightness.

## Front panels

The A-10C writes a full flight-deck state, so any connected front panel renders what it can:

- **FCU / EFIS, PAP3:** heading, indicated airspeed, altitude, and vertical speed.
- **EFIS / FCU baro:** altimeter setting (QNH) from the Kollsman drums, in inHg.
- **AGP32 gear LEDs:** left / nose / right "down & locked"; the in-transit gear-warning
  lamp lights the red UNLK triangles and the lever arrow.
- **AGP32 clock:** the cockpit clock drives the panel — **UTC** is always shown, and when
  the clock is switched to **ET** the elapsed timer is shown as well.

## Notes

- Altitude and barometric pressure are reconstructed from the rotating altimeter drums and
  the Kollsman window, so they track the cockpit gauge rather than a raw number.
