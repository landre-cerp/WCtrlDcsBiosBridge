# F-14B Tomcat

The CDU cycles between two pages: a **RIO CAP** page (tactical data entry) and a **Radio + IFF** page (communications and transponder status). The Radio page is shown on load.

The AGP32 receives gear position lights and the pilot clock/timer.

## Supported devices

| Device | What it shows |
|--------|---------------|
| CDU (MCDU / PFP3N / PFP7 / PFP4) | RIO CAP page, Radio + IFF page |
| AGP32 | Gear position lights (left, nose, right), gear warning, UTC clock, chrono, elapsed time |

## Pages

The CDU switches between two pages using configurable keys (defaults: `PrevPage` → RIO CAP, `NextPage` → Radio+IFF), which you can remap in the Options panel.

The **Radio page** is the default on load.

---

## RIO CAP page

Displays the RIO's Computer Address Panel — the tactical data entry panel used by the Radar Intercept Officer.

| Line | Content | Colour |
|------|---------|--------|
| 0 | `HCU-mode   WCS-mode   CAP-category` | White |
| 1 | *(blank)* | — |
| 2–3 | CAP category label pair 1 | White / Cyan |
| 4–5 | CAP category label pair 2 | White / Cyan |
| 6–7 | CAP category label pair 3 | White / Cyan |
| 8–9 | CAP category label pair 4 | White / Cyan |
| 10–11 | CAP category label pair 5 | White / Cyan |

Each label pair alternates white (upper) / cyan (lower) to match the cockpit CAP panel layout.

### HCU modes (`RIO_HCU_*`)

| Value | Label |
|-------|-------|
| TID | TID display |
| TCS | Television Camera Set |
| RADAR | Radar |
| DDD | Detail Data Display |

### WCS radar modes (`RIO_RADAR_*`)

| Value | Label |
|-------|-------|
| TWS Auto | Track While Scan — automatic |
| Puls Sch | Pulse search |
| TWS Man | Track While Scan — manual |
| PD Sch | Pulse Doppler search |
| PD STT | Pulse Doppler Single Target Track |
| RWS | Range While Search |
| Puls Stt | Pulse Single Target Track |

### CAP categories (`RIO_CAP_CATRGORY`)

| Index | Category |
|-------|----------|
| 0 | BIT — Built-In Test |
| 1 | SPL — Special |
| 2 | NAV — Navigation |
| 3 | TAC DATA — Tactical Data |
| 4 | D/L — Data Link |
| 5 | TGT DATA — Target Data |

> **Note:** `RIO_CAP_CATRGORY` is the exact DCS-BIOS identifier — the typo is in the DCS-BIOS JSON and must be preserved.

## Radio + IFF page

Shows live status for both radios and the IFF transponder.

### UHF — ARC-159 (pilot, lines 0–1)

| Line | Content | Colour |
|------|---------|--------|
| 0 | `UHF  {function}    {freq-mode}` | White |
| 1 | `  {channel}  {frequency}      {SQL}` | Cyan |

- **Function** (`PLT_UHF1_FUNCTION`): `OFF` / `MAIN` / `BOTH` / `ADF`
- **Freq mode** (`PLT_UHF1_FREQ_MODE`): `PRESET` / `MANUAL` / `GUARD`
- **Channel**: `CHxx` (preset number from `PLT_UHF_REMOTE_DISP`) / `MAN` / `GRD`
- **Frequency**: 7-character string from `UHF_FREQ`
- **SQL**: shown when `PLT_UHF1_SQUELCH` is active
- When function is **OFF**, line 1 shows `OFF` instead of frequency data

> **DCS-BIOS quirk:** `PLT_UHF1_FUNCTION` and `PLT_UHF1_FREQ_MODE` report values in reverse order compared to the JSON `positions` array. Labels in the code reflect the actual reported values.

### VUHF — ARC-182 (RIO, lines 3–4)

| Line | Content | Colour |
|------|---------|--------|
| 3 | `VUHF {mode}    {freq-mode}` | White |
| 4 | `  {channel}  {frequency}  {FM/AM}  {SQL}` | Cyan |

- **Mode** (`RIO_VUHF_MODE`): `OFF` / `T/R` / `T/R&G` / `DF` / `TEST`
- **Freq mode** (`RIO_VUHF_FREQ_MODE`): `243` / `MAN` / `G` / `PRESET` / `READ` / `LOAD`
- **Channel**: `CHxx` (numeric from `RIO_VUHF_DISP`) / `243` / `MAN` / `GRD` / `RDG` / `LDG`
- **Frequency**: 7-character string from `VUHF_FREQ`
- **FM/AM** (`RIO_VUHF_FM_AM`): `FM` or `AM`
- **SQL**: shown when `RIO_VUHF_SQUELCH` is active
- When mode is **OFF**, line 4 shows `OFF` instead of frequency data

### IFF — AN/APX-76 (RIO, lines 5–10)

Line 5 is a visual separator (`---...`).

| Line | Content | Colour |
|------|---------|--------|
| 6 | `IFF  MASTER:{master}` | White |
| 7 | `M1:{m1}  M2:{m2}  M3:{m3}` | White |
| 8 | `M4:{m4}  IDT:{ident}  COD:{code}` | White |
| 9 | `  M1:{m1-code}        M3:{m3-code}` | Cyan |

**MASTER** (`RIO_IFF_MASTER`):

| Value | Label |
|-------|-------|
| 0 | OFF |
| 1 | STBY |
| 2 | LOW |
| 3 | NORM |
| 4 | EMER |

**Mode switches** (`RIO_IFF_M1`, `RIO_IFF_M2`, `RIO_IFF_M3A`):

| Value | Label |
|-------|-------|
| 0 | OFF |
| 1 | ON |
| 2 | TST |

**Mode 4** (`RIO_IFF_M4`): `OFF` (0) / `ON` (1)

**IDENT** (`RIO_IFF_IDENT`):

| Value | Label |
|-------|-------|
| 0 | OFF |
| 1 | MIC |
| 2 | IDT |

**CODE** (`RIO_IFF_CODE`):

| Value | Label |
|-------|-------|
| 0 | HLD |
| 1 | A |
| 2 | B |
| 3 | ZRO |

**Codes** (line 9):
- M1 code: two-digit value from `RIO_IFF_M1_10` (10s) + `RIO_IFF_M1_1` (1s)
- M3 code: four-digit value from `RIO_IFF_M3_1000` / `RIO_IFF_M3_100` / `RIO_IFF_M3_10` / `RIO_IFF_M3_1`

Line 11 shows the `(RADIO)` page tag in amber.

## AGP32

### Gear lights

| AGP32 LED | DCS-BIOS control | Meaning |
|-----------|-----------------|---------|
| Gear 1 (left) green | `PLT_GEAR_L_IND_L` | Left main gear down-and-locked |
| Gear 2 (nose) green | `PLT_GEAR_NOSE_IND_L` | Nose gear down-and-locked |
| Gear 3 (right) green | `PLT_GEAR_R_IND_L` | Right main gear down-and-locked |
| UNLK red + red arrow | `PLT_GEAR_L_OFF_L` \| `PLT_GEAR_NOSE_OFF_L` \| `PLT_GEAR_R_OFF_L` | Any gear not locked |

The UNLK warning lights when any of the three OFF flags is active.

### Clock display

| AGP32 display | Content | DCS-BIOS source |
|---------------|---------|-----------------|
| Clock (HH:MM) | UTC hours and minutes | `PLT_CLOCK_H` / `PLT_CLOCK_M` |
| CHR (MM:SS) | Chrono minutes and seconds | `PLT_CLOCK_TM` / `PLT_CLOCK_TS` |
| ET (HH:MM) | Elapsed hours and minutes — shown only when elapsed hours > 0 | `PLT_CLOCK_T` / `PLT_CLOCK_TM` |

All outputs are analog gauge positions scaled via `MaxValue` (`PLT_CLOCK_H` → 0–11 h, `PLT_CLOCK_M` / `PLT_CLOCK_TM` / `PLT_CLOCK_TS` → 0–59, `PLT_CLOCK_T` → 0–11 h). Seconds are fixed at 00 on the main clock (no DCS-BIOS output available).

## Front panels

FCU/EFIS and PAP3 panels stay idle.

## Notes

- The F-14 pilot has no traditional CDU; the CAP is exclusively a RIO instrument. The bridge shows RIO data regardless of which seat is active in DCS.
- Covers both the **F-14B** and **F-14A-135-GR** variants (detected automatically).
