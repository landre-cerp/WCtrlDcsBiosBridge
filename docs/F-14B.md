# F-14B Tomcat (RIO seat)

Displays the RIO's Computer Address Panel (CAP) — the tactical data entry panel used by the
Radar Intercept Officer. The CDU shows the active HCU mode, WCS radar mode, and the ten
label lines of the currently selected CAP category page.

## Supported devices

| Device | What it shows |
|--------|---------------|
| CDU (MCDU / PFP3N / PFP7 / PFP4) | CAP category pages (HCU mode, WCS mode, 5 label pairs) |
| AGP32 | Gear position lights (left, nose, right), UTC clock, chrono timer |

## Pages

The CDU cycles between two pages using configurable keys (defaults: `PrevPage` → RIO CAP, `NextPage` → Radio).
Configure via `useroptions.json`:

```json
"F14RioDisplayKey":   "PrevPage",
"F14RadioDisplayKey": "NextPage"
```

## RIO CAP page

The display is organised as follows:

| Line | Content | Colour |
|------|---------|--------|
| 0 | `HCU-mode   WCS-mode   CAP-category` | White |
| 1 | *(blank)* | — |
| 2–3 | CAP category label pair 1 | White / Cyan |
| 4–5 | CAP category label pair 2 | White / Cyan |
| 6–7 | CAP category label pair 3 | White / Cyan |
| 8–9 | CAP category label pair 4 | White / Cyan |
| 10–11 | CAP category label pair 5 | White / Cyan |

Each label pair uses an alternating colour scheme (white upper line, cyan lower line) to match
the CAP panel layout in the cockpit.

### HCU modes

The Hand Control Unit (`RIO_HCU_*`) selects what the TID (Tactical Information Display) is slaved to:

| Value | Label |
|-------|-------|
| TID | TID display |
| TCS | Television Camera Set |
| RADAR | Radar |
| DDD | Detail Data Display |

### WCS radar modes

The Weapon Control System (`RIO_RADAR_*`) indicates the current radar search/track mode:

| Value | Label |
|-------|-------|
| TWS Auto | Track While Scan — automatic |
| Puls Sch | Pulse search |
| TWS Man | Track While Scan — manual |
| PD Sch | Pulse Doppler search |
| PD STT | Pulse Doppler Single Target Track |
| RWS | Range While Search |
| Puls Stt | Pulse Single Target Track |

### CAP categories (`RIO_CAP_CATRGORY`)

| Index | Category |
|-------|----------|
| 0 | BIT — Built-In Test |
| 1 | SPL — Special |
| 2 | NAV — Navigation |
| 3 | TAC DATA — Tactical Data |
| 4 | D/L — Data Link |
| 5 | TGT DATA — Target Data |

## Brightness / lighting

No dedicated brightness control.

## AGP32

### Gear lights

Each gear leg has two independent DCS-BIOS lights:

| AGP32 LED | DCS-BIOS control | Meaning |
|-----------|-----------------|---------|
| Gear 1 (left) green   | `PLT_GEAR_L_IND_L`    | Left main gear down-and-locked |
| Gear 2 (nose) green   | `PLT_GEAR_NOSE_IND_L` | Nose gear down-and-locked |
| Gear 3 (right) green  | `PLT_GEAR_R_IND_L`    | Right main gear down-and-locked |
| UNLK red + red arrow  | `PLT_GEAR_L_OFF_L` \| `PLT_GEAR_NOSE_OFF_L` \| `PLT_GEAR_R_OFF_L` | Any gear not locked |

The UNLK warning lights when any of the three OFF flags is active. The mechanical drum
indicators (`PLT_GEAR_L_IND` etc.) report a continuous position value and are not used.

### Clock display

<!-- Clock/timer mapping table is defined below (Clock/CHR/ET); removed duplicate table here. -->
`PLT_CLOCK_H` and `PLT_CLOCK_M` are analog gauge position outputs (0–MaxValue); they are
scaled to 0–11 h and 0–59 min respectively. Seconds are fixed at 00 (no DCS-BIOS output
available).

| AGP32 display | Content | DCS-BIOS source |
|---------------|---------|-----------------|
| Clock (HH:MM) | UTC hours and minutes | `PLT_CLOCK_H` / `PLT_CLOCK_M` |
| CHR (MM:SS)   | Chrono minutes and seconds | `PLT_CLOCK_TM` / `PLT_CLOCK_TS` |
| ET (HH:MM)    | Elapsed hours and minutes — shown only when elapsed hours > 0 | `PLT_CLOCK_T` / `PLT_CLOCK_TM` |

All timer outputs are analog gauge positions scaled via MaxValue (`PLT_CLOCK_T` → 0–11 h,
`PLT_CLOCK_TM` → 0–59 min, `PLT_CLOCK_TS` → 0–59 s).

## Radio page

Shows live frequency readouts from both radios (pilot and RIO sides).

| Line | Content | Colour |
|------|---------|--------|
| 0 | `UHF  ARC-159` header | White |
| 1 | Pilot UHF frequency (`PLT_UHF_REMOTE_DISP`) | White |
| 2 | RIO UHF frequency (`RIO_UHF_REMOTE_DISP`) | Cyan |
| 3 | *(blank)* | — |
| 4 | `VUHF ARC-182` header | White |
| 5 | Pilot VHF/UHF frequency (`PLT_VUHF_REMOTE_DISP`) | White |
| 6 | RIO VHF/UHF frequency (`RIO_VUHF_DISP`) | Cyan |
| 11 | `(RADIO)` page tag | Amber |

Frequencies are 7-character strings (e.g. `225.000`) from the radio remote-display outputs.

## Front panels

FCU/EFIS and PAP3 panels stay idle.

## Notes

- The F-14 pilot has no traditional CDU; the CAP is exclusively a RIO instrument.
  The bridge therefore shows the RIO CAP data regardless of which seat is flying.
- `RIO_CAP_CATRGORY` is the DCS-BIOS control identifier — the typo is in the DCS-BIOS JSON
  and must be preserved exactly.
- Covers both the F-14B and F-14A-135-GR variants (detected automatically).
